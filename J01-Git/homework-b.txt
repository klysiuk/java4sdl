Вы разделены на группы.

1. Кто то один из группы создает репозиторий у себя в github аккаунте. Добавляет остальных участников в contributors проекта + меня (lozenko).
2. Группа реализовывает  бранч стратегию user-by-branch, где бранч – это участник проекта. 
3. На первом этапе каждый участник создает три любых текстовых файла, наполняет их каким-либо текстом и сливает с мастером.
4. На втором этапе каждый участник меняет 6 файлов остальных участников и сливает свои изменения с мастером. Эти файлы необходимо взять строго с мастера, а не с бранча другого участника.
Результатом будет состояние мастера, в котором каждый бранч создал текстовые файлы и изменил файлы других. Отсутствие такого результата – 0 баллов каждому участнику.
